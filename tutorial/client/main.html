Inside your main.html file

To begin, we are just going to add a title to our main.html, and nothing else. We will import the rest which will be displayed into our main.js file which again is also located in the client folder. The imports will come from a seperate folder as you'll see next.

Creating your import directory

Create your import directory inside the project folder, and in it create two more directories labeled api and ui. The ui folder will contain the code which will create the body and handle adding and removing tasks on your To-Do List.

Creating the body of your To-Do List

The HTML part of the body contains code which creates the buttons to navigate out of the the page. The body.html file also contains code to create the form and ask the user for input, and also prints out the tasks on to the page when they are created.


Meteor's template element works together with javascript to create dynamic web applications adding new data to the webpage when it is entered. Helpers pass the entered data to the template which then processes it and displays it accordingly. This part of the js file contains a template which defines helper and event functions in javascript. These functions are only called when you press enter, and perform the job of handling the tasks, then sending these tasks over to the html file to be printed to the user.


Creating tasks in your To-Do list

Below is the tasks.html file which contains the code to print the tasks into a field. This code also creates the checkbox which, if checked, will strike out the task indicating it is complete.



The task.js file defines the functions for putting a strike through a completed task when you press the checkbox and adding a button to delete the task.


The tasks.js file contains the code to create the collection, which stores the tasks on a database and is available to both the client and the server side to use. Every time an event is called, it does a Task.remove or Task.update which are both part of the MongoDB database. The code below creates a MongoDB collections object which has already defined the functions to update and remove and we can use these functions with a simple function call.
